Ingestion Specifications & Data Rules

Rules and structures applied to batch JSON ingestion and database ETL sync.

Pipeline A: Customer JSON Ingestion Rules

Processes batch JSON customer records. Expected keys are mapped dynamically through self-healing configs.

customer_id TEXT (PRIMARY KEY) • Required

Must not be null or empty. Coerced to string (e.g. 5 becomes "5"). Duplicate entries in batch or DB are quarantined.

email TEXT • Required

Must not be null or empty. Must contain "@" character. Coerced to string. Malformed values are quarantined.

name / country TEXT • Optional

Coerced to string. Defaults to empty string ("") if missing or null.

Pipeline B: Device Telemetry ETL Rules

Processes database-to-database aggregations from device_telemetry to system_performance_metrics.

device_id TEXT (PRIMARY KEY) • Required

Must not be null or empty. Aggregation rows with null device_id are isolated to the quarantine log.

total_events INTEGER • Calculated

Total log events registered for the given device.

error_rate / avg_response_time REAL • Calculated

Error rate calculation and average query response time aggregate.
